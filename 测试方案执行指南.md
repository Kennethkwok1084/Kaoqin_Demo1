# 考勤管理系统 - 测试方案执行指南

## 📋 项目测试概述

本项目是一个基于 **FastAPI + Vue3** 的考勤管理系统，拥有完善的测试框架配置：
- **后端**：pytest + pytest-asyncio + SQLAlchemy测试 + 覆盖率报告
- **前端**：Vitest (单元测试) + Playwright (E2E测试)
- **目标覆盖率**：项目40%+，新代码50%+

## 🚀 快速开始

### 1. 环境准备

```bash
# 后端测试环境
cd backend
python -m venv .venv
source .venv/bin/activate  # Linux/Mac
# .venv\Scripts\activate   # Windows
pip install -r requirements-dev.txt

# 前端测试环境
cd frontend
npm install
```

### 2. 数据库启动

```bash
# 启动测试数据库
docker-compose up -d postgres redis

# 运行数据库迁移
cd backend && alembic upgrade head
```

### 3. 验证测试环境

```bash
# 后端基础验证
cd backend && pytest tests/test_basic.py -v

# 前端基础验证
cd frontend && npm run test:unit tests/unit/minimal.test.ts
```

## 🎯 测试最佳实践

### 1. 测试编写规范
- **命名**: `test_功能描述_期望结果`
- **结构**: Arrange-Act-Assert (AAA) 模式
- **独立性**: 每个测试独立，不依赖执行顺序

### 2. 数据准备策略
- 使用 **Factory Boy** 创建测试数据
- 使用 **fixtures** 管理测试状态
- 测试后自动清理数据

### 3. Mock和存根
```python
# 后端Mock示例
from unittest.mock import patch

@patch('app.services.email_service.send_email')
def test_notification(mock_send):
    # 测试逻辑
    pass
```

```javascript
// 前端Mock示例
import { vi } from 'vitest'

const mockApi = vi.mocked(api.getUsers)
mockApi.mockResolvedValue([])
```

### 4. 异步测试处理
```python
# pytest-asyncio
@pytest.mark.asyncio
async def test_async_function():
    result = await async_function()
    assert result is not None
```

## 📈 CI/CD 集成

### GitHub Actions 示例
```yaml
name: Test Suite
on: [push, pull_request]

jobs:
  backend-tests:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:15
        env:
          POSTGRES_PASSWORD: testpass
    
    steps:
      - uses: actions/checkout@v4
      - name: Setup Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.12'
      
      - name: Install dependencies
        run: |
          cd backend
          pip install -r requirements-dev.txt
      
      - name: Run tests
        run: |
          cd backend
          pytest --cov=app --cov-report=xml
      
      - name: Upload coverage
        uses: codecov/codecov-action@v3

  frontend-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'
      
      - name: Install dependencies
        run: |
          cd frontend
          npm ci
      
      - name: Run tests
        run: |
          cd frontend
          npm run test:unit:coverage
          npm run test:e2e
```

## 🎪 完整测试流程示例

### 开发新功能的测试流程
```bash
# 1. 创建功能分支
git checkout -b feature/new-attendance-rule

# 2. 编写后端测试
cd backend
pytest tests/unit/test_new_feature.py -v

# 3. 编写前端测试
cd frontend
npm run test:unit -- tests/unit/components/NewFeature.test.ts

# 4. 运行完整测试套件
cd backend && pytest -m "unit or integration" --cov=app
cd frontend && npm run test:all

# 5. 检查覆盖率
cd backend && pytest --cov=app --cov-report=html
cd frontend && npm run test:unit:coverage

# 6. 提交前最终验证
./deploy_ready_check.sh
```

### 发布前完整验证
```bash
# 1. 完整后端测试
cd backend
pytest --cov=app --cov-report=term-missing

# 2. 完整前端测试
cd frontend
npm run test:all

# 3. 系统集成测试
```
