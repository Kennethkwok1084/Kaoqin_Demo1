# 考勤管理系统业务测试方案

## 1. 测试计划书

### 1.1 测试范围
本测试方案覆盖《考勤管理系统》后端核心业务逻辑与前端关键交互，具体包含：
- **计算逻辑**：工时计算（基础、奖励、惩罚）、月度汇总、结转逻辑。
- **算法逻辑**：报修单A/B表匹配算法、Excel解析算法。
- **业务逻辑**：任务全生命周期管理（报修、监控、协助）、审批流、权限控制。

### 1.2 测试策略
- **分层测试**：
    - **单元测试 (Unit)**：针对计算函数、匹配算法进行独立验证。
    - **集成测试 (Integration)**：验证Service层与Database层的交互，确保事务一致性。
    - **端到端测试 (E2E)**：模拟用户从登录到任务处理的完整流程。
- **数据驱动**：使用边界值、异常值驱动测试用例设计。
- **自动化优先**：核心业务逻辑通过 Pytest 实现自动化回归。

### 1.3 资源计划
- **测试环境**：预发布环境 (Pre-prod)，配置独立的 PostgreSQL 和 Redis 实例。
- **工具链**：Pytest (后端自动化), Playwright (前端E2E), Allure (测试报告)。

### 1.4 风险识别
- **算法准确性**：姓名/电话匹配可能存在同名或格式不规范导致的误判。
- **并发一致性**：高并发下工时统计可能出现脏读或更新丢失。
- **规则变更**：爆单规则冻结等业务调整需确保代码与文档一致。
