## 3. 缺陷与问题记录 (Defect Report)

本次测试主要发现环境配置与代码兼容性问题，业务逻辑无明显缺陷。

| ID | 问题描述 | 严重程度 | 状态 | 根因分析 | 修复方案 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| DEF-001 | `.env` 配置格式错误导致 Pydantic 解析失败 | High (Blocker) | 已修复 | Pydantic v2 对 List 类型校验严格，`.env` 中使用逗号分隔字符串而非 JSON 列表。 | 更新 `.env` 中的 `PWD_CONTEXT_SCHEMES` 和 `ALLOWED_FILE_TYPES` 为 JSON 格式。 |
| DEF-002 | 测试环境数据库连接失败 | High (Blocker) | 已修复 | 本地无运行中的 PostgreSQL 实例，测试配置默认连接 PG。 | 配置 `FORCE_SQLITE_TESTS` 并修补 `database_config.py` 与 `database.py` 以支持 SQLite 模式。 |
| DEF-003 | 异步测试中 Lazy Load 触发 `MissingGreenlet` | Medium | 已修复 | 在 `WorkHoursService` 中访问 `task.tags` 时，由于 Session 提交后对象 Detach，导致隐式 IO 调用失败。 | 在测试用例中显式使用 `selectinload` 重新加载关联数据。 |
